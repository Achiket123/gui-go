Overview

gui-go is a native GUI framework written entirely in Go. It uses OpenGL for rendering (via CGo → X11 on Linux) and provides a complete widget toolkit, reactive state management, an animation engine, and built-in dev tools — all in one repository with zero external Go dependencies.

ℹ️

Every object on screen implements the four-method ui.Component interface. This tiny contract is how the entire framework composes — layouts wrap components, dev tools wrap components, screens navigate between components.

Package Map

PackageRole
gouiRoot — NewWindow, main run loop
uiAll 20+ widgets (Button, Input, Modal, Toast, Navbar, …)
canvas2-D drawing API: rect, circle, text, polygon, clip, gradient
stateSignal[T], Computed[T], Store[S], History[T], EventBus
animationAnimationController, easing, spring physics
themeColor palette, typography scale, radius tokens
render/glOpenGL batch renderer — Batch, VAO, VBO, shader
platformX11 CGo bindings — window lifecycle, events, pixmap
devtoolsLayoutDebugger, FPSOverlay, EventLogger, HotReloader, Inspector

Design Philosophy

gui-go is retained-mode: every frame redraws the entire scene. There is no virtual-dom, no dirty-checking, and no widget tree reconciliation. Components accumulate draw calls into a GPU batch during their Draw method; at end-of-frame the batch is flushed in a single glDrawElements call. This keeps the mental model simple: your code runs top-to-bottom, every frame.

Quick Start

The minimal app: clear the canvas, add a reactive label and an increment button.

go
package main

import (
  goui "github.com/achiket123/gui-go"
  "github.com/achiket123/gui-go/canvas"
  "github.com/achiket123/gui-go/state"
  "github.com/achiket123/gui-go/ui"
)

func main() {
  count := state.New(0)
  label := ui.NewLabel("Count: 0")

  // React to signal changes
  count.Subscribe(func(v int) {
    label.Text = fmt.Sprintf("Count: %d", v)
  })

  btn := ui.NewButton("+ Increment", func() {
    count.Set(count.Get() + 1)
  })

  win := goui.NewWindow("Counter", 480, 320)
  // ❶ Always clear first — prevents GPU frame residue
  win.AddComponent(ui.NewClearRect(canvas.Hex("#111420")))
  // ❷ FixedPosition forces explicit pixel rect
  win.AddComponent(ui.NewFixedPosition(label, 40, 80, 320, 40))
  win.AddComponent(ui.NewFixedPosition(btn,   40, 140, 140, 40))
  win.Run()
}
💡

Always add a ClearRect as the very first component. The GPU does not clear between frames — without it you get ghosting from the previous frame.

Frame Loop

Each frame, the window does exactly five things in order:

  1. Poll events — platform.NextEvent drains the X11 queue; each event dispatches to HandleEvent on the root component tree (first consumer wins).
  2. Tick — Tick(delta) is called on every component. Animations advance here; delta is seconds since the previous frame.
  3. Reset batch — the GL batch is cleared for the new frame.
  4. Draw — Draw(c, x, y, w, h) is called on every component top-to-bottom. Components push quads/glyphs into the batch via canvas.Canvas.
  5. Flush — one glDrawElements uploads and draws the entire batch. Back buffer swaps.

Component Interface

Everything visible implements exactly four methods:

go
type Component interface {
  // Bounds returns the axis-aligned rect from the last Draw call.
  // Used for hit-testing and by the LayoutDebugger overlay.
  Bounds() canvas.Rect

  // Tick advances internal state (animations, timers, blinking cursors).
  // delta is seconds elapsed since the previous frame (usually ~0.016).
  Tick(delta float64)

  // HandleEvent processes one input event.
  // Return true to consume the event — propagation stops.
  HandleEvent(e Event) bool

  // Draw renders into canvas c at the given rect.
  // Called every frame. Store bounds here for hit-testing.
  Draw(c *canvas.Canvas, x, y, w, h float32)
}

Implement these four methods and your type is a first-class gui-go component — wrap it in a layout, pass it to a ModalManager, register it with the LayoutDebugger, or navigate to it as a screen.

Events flow down the component tree in registration order. The first HandleEvent that returns true consumes the event — subsequent components don't see it.

Hit Testing Pattern

go
func (w *MyWidget) HandleEvent(e ui.Event) bool {
  r := w.bounds
  inside := e.X >= r.X && e.X <= r.X+r.W &&
             e.Y >= r.Y && e.Y <= r.Y+r.H

  switch e.Type {
  case ui.EventMouseMove:
    w.hovered = inside
    return inside   // consume only when inside
  case ui.EventMouseDown:
    if inside && e.Button == 1 {
      w.onClick()
      return true
    }
  case ui.EventKeyDown:
    switch e.Key {
    case "ArrowUp", "+":  w.increment(); return true
    case "ArrowDown", "-": w.decrement(); return true
    }
  }
  return false
}

Signal & Store

Signal[T]

Thread-safe generic reactive value. Notifies subscribers only when the value actually changes (equality check). Use NewSignalAny[T] for non-comparable types.

go
// Create
count := state.New(0)         // *Signal[int]
name  := state.New("Alice")  // *Signal[string]

// Read (thread-safe)
v := count.Get()

// Write (notifies only if value changed)
count.Set(42)

// Atomic transform
count.Update(func(n int) int { return n + 1 })

// Subscribe — returns unsubscribe function
stop := count.Subscribe(func(v int) {
  label.Text = fmt.Sprintf("Count: %d", v)
})
defer stop()  // clean up

// OnChange alias
count.OnChange(func(v int) { /* same as Subscribe */ })

Store[S]

go
type AppState struct { Count int; Name string }

store := state.NewStore(AppState{Name: "hello"})

// Mutate (runs under lock, then notifies)
store.Mutate(func(s *AppState) {
  s.Count++
  s.Name = "world"
})

snap := store.Get()           // AppState value copy

store.Subscribe(func(s AppState) {
  fmt.Println("new state:", s)
})

Computed[T]

go
first := state.New("Jane")
last  := state.New("Doe")

fullName := state.Derive(func() string {
  return first.Get() + " " + last.Get()

Watch (Effect)

go

History — Undo / Redo

go
count   := state.New(0)
history := state.NewHistory(0, 50)  // initial=0, maxDepth=50

// Record before mutating
history.Push(count.Get() + 1)
count.Set(history.Get())

// Undo (returns false if stack empty)
if history.Undo() {
  count.Set(history.Get())
}

// Redo
if history.Redo() {
  count.Set(history.Get())
}

// Access underlying Signal
history.Signal().Subscribe(func(v int) { /* … */ })
💡

The counter example wires undo to both the "Undo" button and keyboard shortcut Z / Ctrl+Z in the HandleEvent switch — full keyboard navigation with undo history in under 40 lines.

EventBus

Topic-based, type-safe publish-subscribe bus for decoupled communication between unrelated components.

go
bus := state.NewEventBus()

// Type-safe subscribe
sub := state.On[string](bus, "user:login", func(user string) {
  fmt.Println("logged in:", user)
})
defer sub.Cancel()

// Type-safe publish
state.Publish[string](bus, "user:login", "alice")

// Raw (untyped) variants
bus.RawPublish("nav:back", nil)
bus.RawSubscribe("nav:back", func(any) { /* … */ })

Animation System

gui-go's animation system is tick-driven: no goroutines, no timers. Each component's Tick(delta) advances its controllers, and Draw reads the current value to influence rendering.

go
// Create a 600 ms controller
anim := animation.NewController(600 * time.Millisecond)
anim.Easing = animation.EaseOutCubic

// Control direction
anim.Forward()   // play 0→1
anim.Backward()  // play 1→0
anim.Toggle()    // flip direction

// In Tick(delta)
anim.Tick(delta)

// In Draw()
t := float32(anim.Value())  // 0.0 – 1.0, eased

// Example — fade-in card
bg := canvas.Hex("#1e2335")
bg.A = t
c.DrawRoundedRect(cx, cy, cw, ch, 12, canvas.FillPaint(bg))

// Example — grow bar chart
barH := maxH * t
c.DrawRoundedRect(bx, y+h-barH, bw, barH, 3, canvas.FillPaint(col))

GL Renderer

The render/gl package implements a deferred batch renderer. All canvas draw calls push quads into a Batch; at end-of-frame the batch is uploaded and drawn in one glDrawElements.

MethodDescription
PushQuad(x0,y0,x1,y1, uv, rgba, mode…)Add a textured/colored quad (4 verts, 2 triangles)
PushTriangle(x0,y0, x1,y1, x2,y2, rgba, mode)Bare triangle — polygon fans, custom shapes
NeedsFlush()Returns true when approaching 65536 vertex limit
Flush()Upload to GPU + glDrawElements — resets batch
Reset()Clear without uploading (start of new frame)
Destroy()Free VAO, VBO, EBO GPU resources
ℹ️

Max capacity is 65 536 vertices (uint16 index range). The renderer auto-flushes mid-frame when NeedsFlush() is true, so UIs with thousands of primitives still render correctly without manual batching.

Dev Tools

All tools are in the devtools package and compiled only with //go:build debug. Zero production overhead.

LayoutDebugger

Wraps a root component and overlays colored dashed borders around every registered component's Bounds(). Toggle with F1.

go
dbg := devtools.NewLayoutDebugger(rootWidget)
dbg.Register(counterWidget)
dbg.Register(buttonBar)
// Press F1 in app to toggle the overlay on/off

FPSOverlay

go
fps := devtools.NewFPSOverlay()
fps.Corner  = 3    // 0=TL 1=TR 2=BL 3=BR
fps.BgAlpha = 0.7
fps.RecordFrame(delta)  // in render loop
fps.Draw(c, x, y, w, h)

HotReloader

go
rl := devtools.NewHotReloader(".", func() {
  log.Println("restarting…")
  os.Exit(0)
})
rl.BuildCmd = []string{"go","build","-tags","debug","-o",os.Args[0],"."}
go rl.Watch()  // non-blocking goroutine

Platform / X11

The platform package provides thin CGo wrappers over Xlib. Normally you never call these directly — goui.NewWindow handles everything. They're useful for advanced embedding or custom window management.

FunctionDescription
OpenDisplay() / CloseDisplay()Connect / disconnect from X server
CreateSimpleWindow(…)Create window, returns X11 window ID
MapWindow(dpy, win)Make window visible on screen
SelectInput(dpy, win, mask)Register event mask (KeyPress, ButtonPress, …)
NextEvent(dpy)Dequeue next event → decoded XEventData struct
CreatePixmap(…)Offscreen buffer for double-buffering
CreateXImage(…)BGRX XImage backed by a Go byte slice
PutXImage(…)Blit pixel buffer to a drawable
Flush(dpy) / Sync(dpy)Flush / await X server command queue
InternAtom(dpy, name, …)Look up X11 atom ID by name
SetWMProtocols(dpy, win, atoms)Register WM protocols e.g. WM_DELETE_WINDOW
